# AI 小说创作助手 (AI Novelist Assistant)

这是一个基于 React 和 Gemini API 构建的专业小说创作辅助工具。它旨在通过多 Agent 协作和 RAG（检索增强生成）技术，帮助创作者更高效地完成从构思到成稿的全过程。

## 核心功能

### 1. 多 Agent 协作流 (Multi-Agent Workflow)
将创作过程拆分为四个关键阶段，每个阶段由专门的 AI 角色负责：
- **阶段 0：剧情构思讨论** - 在生成大纲前，与 AI 深入探讨本章的想法、冲突和走向。
- **阶段 1：剧情大纲策划** - 根据讨论记录和前情提要，自动生成详细的章节大纲。
- **阶段 2：正文扩写主笔** - 将大纲扩写为生动的小说正文，保持设定的文风。
- **阶段 3：细节精修润色** - 对草稿进行最后的打磨，提升表现力并消除语病。

### 2. 智能大纲推演 (Brainstorming)
内置“推演”功能，模拟多位专家（架构师、创意大师、细节控）进行多轮对话，帮助你突破创作瓶颈，生成连贯的剧情方案。

### 3. 角色与设定管理
- **角色库**：管理人物姓名、身份、性格及详细设定。支持从正文或摘要中一键提取角色。
- **设定集 (Lorebook)**：记录世界观、地点、道具、功法等关键设定。AI 在创作时会严格遵循这些设定。

### 4. RAG 增强检索
支持构建本地向量知识库。AI 助手可以实时检索全书已有的章节内容，确保长篇创作中的逻辑一致性和细节准确性。

### 5. 灵活的 AI 配置
- 支持 **Gemini 3 Flash** 和 **Gemini 3.1 Pro**。
- 支持自定义 OpenAI 兼容接口。
- 可配置温度、文风参考范文、负面提示词等。

## 技术栈
- **前端**：React, TypeScript, Tailwind CSS
- **图标**：Lucide React
- **动画**：Framer Motion (motion/react)
- **AI SDK**：@google/genai
- **向量检索**：基于本地内存的简单向量实现 (RagService)

## 快速开始

### 1. 环境准备
确保你的电脑已安装 [Node.js](https://nodejs.org/) (建议 v18 或更高版本)。

### 2. 安装依赖
在项目根目录下运行：
```bash
npm install
```

### 3. 启动项目
运行以下命令启动开发服务器：
```bash
npm run dev
```
项目启动后，在浏览器中访问 `http://localhost:3000` 即可开始使用。

## 使用流程

1. **配置 AI 引擎**：
   - 点击左上角的“设置”图标。
   - 输入你的 Gemini API Key（或配置 OpenAI 兼容的自定义接口）。
   - 选择你偏好的 AI 模型（推荐使用 Gemini 3.1 Pro 进行深度创作）。

2. **设定小说基调**：
   - 在左侧栏点击小说标题，设定全局文风、参考范文和负面提示词（禁止项）。
   - 这些设定将作为 AI 创作的“宪法”，确保文风统一。

3. **完善世界观**：
   - 在“角色”和“设定”选项卡中添加关键人物和背景设定。
   - 你也可以先写一段正文，然后使用“一键提取”功能让 AI 自动帮你完善设定集。

4. **开启多 Agent 协作**：
   - 进入“多 Agent 工作流”面板。
   - **讨论**：先告诉 AI 你这一章想写什么。
   - **大纲**：让 AI 生成详细的章节大纲。
   - **正文**：AI 根据大纲和设定集扩写初稿。
   - **润色**：对初稿进行文学性加工。

5. **利用 RAG 保持逻辑一致**：
   - 当小说字数较多时，点击“构建知识库”。
   - AI 会在对话时自动检索前文细节，避免出现“吃设定”的情况。

## 响应式支持
本应用已适配 **电脑、平板和手机**。
- **电脑端**：三栏布局，视野最广。
- **平板端**：自动折叠侧边栏，聚焦创作。
- **手机端**：底部导航切换，随时随地记录灵感。
